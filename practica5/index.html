Pagina Web.

Rojas Robles Iván
Ingenieria Mecatrónica
Fecha: Marzo 14, 2025
Programación GitHub

- Resumen -

Este proyecto consiste en la creación de una página web utilizando GitHub como plataforma de hosting. Se detallan los pasos para la creación de una cuenta, la configuración de un repositorio y la activación de la URL pública.

Se implementa un código que permite la navegación entre diferentes secciones de la página sin necesidad de salir de la principal.

- Introducción -

El uso de GitHub para el desarrollo y despliegue de páginas web es una alternativa eficiente y gratuita para proyectos personales y colaborativos.

A través de GitHub Pages, es posible alojar sitios estáticos de manera sencilla. En este documento se explica el proceso de configuración y se presenta un diseño funcional con navegación interna optimizada.

Cuenta de GitHub

- Desarrollo -

Creación de cuenta en GitHub: Ingresar a GitHub y registrarse si no se tiene cuenta. Creación de un repositorio: Acceder a GitHub y hacer clic en "New Repository". Asignar un nombre al repositorio y seleccionar la opción "Público". Inicializar con un archivo README.md (opcional). Activación de GitHub Pages: Ir a la pestaña "Settings" del repositorio. En la sección "Pages", seleccionar la rama principal como fuente de publicación. Guardar los cambios y esperar a que la URL de la página se active. Subida de archivos y configuración inicial: Crear archivos HTML en el repositorio. Subir o sincronizar los archivos usando Git o la interfaz web.

Simulación

Se probará la funcionalidad del sitio web accediendo a la URL generada por GitHub Pages. Se verificará la navegación entre páginas sin necesidad de recargar la página principal.

- Resultados -

Se logró la configuración y activación de GitHub Pages. Se implementó un diseño funcional con navegación interna. Se verificó la correcta carga de las secciones sin salir de la página principal.

- Conclusiones -

GitHub Pages es una herramienta efectiva para el alojamiento de sitios web estáticos. Con una configuración adecuada, es posible desarrollar páginas interactivas y dinámicas sin necesidad de servidores externos. La implementación de navegación interna mejora la experiencia del usuario y optimiza el rendimiento del sitio.
